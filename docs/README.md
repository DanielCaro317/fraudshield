# 📁 docs — Documentación del proyecto FraudShield

Documentación, planeación y material de preparación del proyecto. El código y los manuales técnicos viven en las carpetas del proyecto (`manuales/`, `dbt_project/`, `rag/`, etc.); aquí guardamos lo estratégico y personal.

## Contenido

### 📘 `manuales_word/`
Los manuales paso a paso en formato **Word (.docx)**, generados desde la fuente en Markdown (`../manuales/`). Formato doble camino **🖱️ UI + ⌨️ CLI**. Para leer/imprimir/compartir cómodo.

> 💡 La **fuente** editable de estos manuales está en [`../manuales/`](../manuales/) (Markdown). Cuando se actualiza un manual, se regenera su `.docx` aquí.

### 🎯 `planeacion/`
- `PLAN_APRENDIZAJE_CEIBA_ML.md` — plan de aprendizaje para el cargo de Ingeniero ML en **Ceiba**.
- `PLAN_APRENDIZAJE_PROXIFY_AI_ENGINEER.md` — diagnóstico + plan + mapa de fusión para **Proxify** (Senior AI Engineer).
- `PREPARACION_ENTREVISTA_TECNICA_PROXIFY.md` — banco de preguntas y guiones para la entrevista técnica de Proxify.
- `job.md` — descripción del cargo de Ceiba (referencia de requisitos).

### 🗣️ `entrevista_proxify/`
- `transcripcion_entrevista_rrhh.md` — transcripción de la entrevista de RR.HH. con Proxify (Mariana). Útil para revisar tono, gaps detectados y expectativas.
- `prep_tecnica/` — material de preparación técnica rescatado (planes, setup, milestones, technical prep).
